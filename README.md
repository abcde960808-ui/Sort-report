## 一、 複雜度分析對照表

| 排序法 | 平均時間複雜度 | 最差時間複雜度 | 空間複雜度 | 穩定性 |
| :--- | :--- | :--- | :--- | :--- |
| **氣泡排序** | O(n²) | O(n²) | O(1) | 穩定 |
| **選擇排序** | O(n²) | O(n²) | O(1) | 不穩定 |
| **插入排序** | O(n²) | O(n²) | O(1) | 穩定 |
| **合併排序** | O(n log n) | O(n log n) | O(n) | 穩定 |
| **快速排序** | O(n log n) | O(n²) | O(log n) | 不穩定 |

## 二、 數據模擬結果 (實驗分析)

透過實測不同資料量 (N) 下各演算法所需的執行時間（模擬數據）：

| 數據規模 (N) | Bubble | Selection | Insertion | Merge | Quick |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **100** | ~1ms | ~1ms | ~1ms | <1ms | <1ms |
| **5,000** | 約 45ms | 約 30ms | 約 20ms | <5ms | <5ms |
| **10,000** | 約 180ms | 約 120ms | 約 80ms | 約 8ms | 約 6ms |

### 實驗觀察
隨著數據量 N 的增加，**O(n²)** 類型的排序法（Bubble, Selection, Insertion）執行時間呈平方級別成長；而 **O(n log n)** 類型（Merge, Quick）則展現出極高的效率優勢。
